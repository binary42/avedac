Event Table Operations: Delete, Combine, Unselect.

Once events are selected,  a few operations can be performed on these events, namely deleting, combining, and unselecting.  The algorithm for automated event detection and tracking does product false detections and tracking can skip frames for many reasons beyond the scope of this help document.  To assist with cleaning up these errors, the capability to delete and combine events exists in the user interface.

If one of more events are selected, selecting delete will delete the event reference.  This might be used, for example, to delete false detections. This can also be useful to simply remove events that are not of interest to you.

Combing events is useful for cases when the same object/animal is detected multiple times as a separate event.  The combine operation will combine all events according to the algorithm:

Using the lowest objectId of the selction (which is completely arbitrary and has no significance), combine events.  If two events exist in the same frame, the conflict is  resolved by simply choosing the event with the larger square pixesl area.  This proceeds in contiguous order by object ID.

Unselecting events  simply unselects the user selection.

Once events are selected, delete events with Control-Click, then select Delete, or Delete all (if multiple events selected).
